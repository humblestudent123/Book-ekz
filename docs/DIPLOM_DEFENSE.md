# Сценарий защиты: автоматизация развертывания сервиса рекомендаций

Цель: показать **воспроизводимое развертывание** (Docker Compose), **ускорение ответов** (Redis) и **контроль качества** (GitLab CI).

## 1. Архитектура (1–2 минуты)

- Пользователь открывает React-приложение (библиотека ReadNext).
- При загрузке страница «Рекомендации» запрашивает `POST /api/recommendations` с ID книг из **localStorage** (избранное).
- Сервис `recommendation-api` (Express) считает подбор по жанрам/тегам и отдаёт также блок **курсов**.
- Повторные запросы с тем же избранным обслуживаются из **Redis** (поле `meta.cache: HIT`).

Схема в репозитории: `docker-compose.yml` (сервисы `web`, `api`, `redis`).

## 2. Демонстрация Docker Compose (2–3 минуты)

В корне проекта:

```bash
docker compose up --build
```

Далее в браузере:

1. `http://localhost:3000/library` — убедиться, что каталог открывается.
2. В блоке «Рекомендации» прочитать подпись: первый ответ — **с сервера**, после обновления страницы при активном Redis — **из кэша**.
3. При необходимости добавить книгу в избранное на странице книги, вернуться в библиотеку — список рекомендаций меняется.

Проверка API отдельно: `http://localhost:4000/health`.

## 3. Демонстрация GitLab CI (1–2 минуты)

В репозитории настроен файл `.gitlab-ci.yml`:

- установка зависимостей фронта и API;
- `npm test` в `recommendation-api`;
- `npm run test:ci` для CRA;
- `npm run build`;
- job `smoke_api`: подъём API с `SKIP_REDIS=1` и проверка `/health`.

Показать **зелёный pipeline** в интерфейсе GitLab.

## 4. Вопросы комиссии (краткие ответы)

| Вопрос | Ответ |
|--------|--------|
| Зачем Redis? | Кэшировать результат рекомендаций для одинаковых запросов и снизить время ответа. |
| Что если Redis недоступен? | API продолжает работать, кэш отключается, в `/health` отражается состояние. |
| Почему Docker? | Одинаковое окружение на машине преподавателя и у разработчика, простой запуск всего стенда одной командой. |
| Насколько «умные» рекомендации? | Дипломный упрощённый скоринг по пересечению жанров/тегов с избранным + буст новинок/популярного; фокус защиты — **DevOps**, не ML. |
