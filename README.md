# ReadNext / Book-ekz

## Дипломный frontend-проект: система поиска и рекомендаций книг

ReadNext — это одностраничное веб-приложение для просмотра каталога книг, поиска, фильтрации по жанрам, открытия страницы конкретной книги и чтения текста внутри модального окна. Проект подготовлен как практическая часть дипломной работы и показывает полный пользовательский сценарий: от выбора книги в библиотеке до чтения и возврата обратно в каталог.

Проект можно запустить на обычном компьютере без специальных знаний в программировании. Ниже есть подробная инструкция: что установить, какие кнопки нажимать, какие команды вводить и как понять, что всё работает правильно.

---

## Кратко о проекте

**Название проекта:** Book-ekz / ReadNext  
**Тип проекта:** frontend-приложение  
**Основная технология:** React  
**Сборка и запуск:** react-scripts  
**Язык интерфейса:** русский  
**Хранение данных:** локальные файлы проекта и localStorage браузера  
**Backend-сервер:** не требуется  
**База данных:** не требуется  

Приложение работает в браузере. После запуска пользователь открывает страницу библиотеки, видит карточки книг, может искать книгу по названию или автору, выбрать жанр, открыть страницу книги, нажать кнопку чтения, листать страницы и закрыть читалку.

---

## Для чего нужен проект

Цель проекта — реализовать удобный интерфейс электронной библиотеки, в котором пользователь может:

1. Просматривать каталог книг.
2. Быстро находить нужную книгу через поиск.
3. Фильтровать книги по жанрам.
4. Открывать отдельную страницу книги.
5. Читать текст книги прямо в браузере.
6. Листать страницы в модальном окне чтения.
7. Добавлять книги в избранное.
8. Возвращаться из страницы книги обратно в библиотеку.
9. Продолжать пользоваться приложением без регистрации и без подключения к серверу.

Проект демонстрирует навыки разработки SPA-приложения на React, работу с компонентами, маршрутизацией, локальными данными, пользовательским состоянием и E2E-тестированием.

## Основные возможности приложения

### 1. Библиотека

Главная страница приложения находится по адресу:

```text
http://localhost:3000/library
```

На ней отображаются:

- логотип ReadNext;
- строка поиска;
- фильтр по жанрам;
- блоки с книгами;
- карточки книг;
- обложка, название, автор, год и жанры каждой книги.

Карточки книг имеют класс:

```text
.book-card
```

Это важно для тестирования и проверки интерфейса.

### 2. Поиск

Пользователь может ввести название книги, имя автора, год или жанр в строку поиска. Каталог автоматически отфильтрует книги по введённому тексту.

### 3. Фильтр по жанрам

Справа над каталогом расположен выпадающий список жанров. В нём есть пункт:

```text
Все жанры
```

Если выбран этот пункт, отображается весь каталог. Если выбран конкретный жанр, отображаются только книги этого жанра.

### 4. Страница книги

При клике на карточку книги пользователь переходит на страницу вида:

```text
http://localhost:3000/library/book/5
```

На странице книги отображаются:

- большая обложка;
- название;
- автор;
- год;
- жанры;
- описание;
- кнопка чтения;
- кнопка добавления в избранное.

### 5. Читалка ReaderModal

Кнопка чтения открывает модальное окно ReaderModal. В нём пользователь может:

- читать текст книги;
- перейти на следующую страницу;
- вернуться на предыдущую страницу;
- закрыть модальное окно;
- использовать клавиши стрелок на клавиатуре;
- закрыть окно клавишей Esc.

Для E2E-тестов используются стабильные селекторы:

```text
data-testid="open-reader"
data-testid="next-page"
data-testid="prev-page"
data-testid="close-reader"
```

### 6. Сохранение состояния

Приложение использует localStorage браузера. Это значит, что некоторые данные сохраняются прямо в браузере пользователя:

- избранные книги;
- текущая страница чтения;
- статистика открытия книг;
- цитаты, если пользователь выделяет текст и сохраняет его.

Если очистить данные сайта в браузере, эти сохранённые данные будут удалены.

## Что нужно установить перед запуском

Для запуска проекта нужен Node.js. Это программа, которая позволяет запускать JavaScript-проекты на компьютере.

### 1. Установить Node.js

1. Откройте сайт:

```text
https://nodejs.org/
```

2. Скачайте версию **LTS**.
3. Запустите установщик.
4. Нажимайте **Next** до конца установки.
5. После установки перезагрузите терминал, если он был открыт.

### 2. Проверить, что Node.js установился

Откройте терминал:

- на Windows можно открыть **PowerShell**;
- можно открыть терминал прямо в папке проекта;
- можно нажать в проводнике в адресную строку, написать `cmd` и нажать Enter.

Введите:

```bash
node -v
```

Если всё хорошо, появится версия, например:

```text
v22.17.1
```

Потом проверьте npm:

```bash
npm -v
```

Если появилась версия npm, значит базовая установка готова.

В папке проекта выполните:

```bash
npm install
```

Что делает эта команда:

- читает файл `package.json`;
- скачивает нужные библиотеки;
- создаёт папку `node_modules`;
- подготавливает проект к запуску.

## Тестирование проекта

В проекте есть два вида E2E-тестирования:

1. Cypress.
2. Playwright.

E2E означает end-to-end. Такие тесты имитируют действия реального пользователя в браузере: открыть страницу, нажать на карточку, выбрать жанр, открыть читалку, листать страницы и закрыть окно.

### Где лежат тесты

```text
tests/
  cypress/
    cypress.config.js
    e2e/
      library.cy.js
      bookPage.cy.js
      readerModal.cy.js

  playwright/
    playwright.config.js
    e2e/
      library.spec.js
      bookPage.spec.js
      readerModal.spec.js
```

### Первый запуск тестов

Перед первым запуском можно установить браузеры и бинарники тестовых инструментов:

```bash
npm run cypress:install
npm run playwright:install
```

Обычно часть этого скачивается автоматически, но эти команды полезны для чистой установки.

### Запуск всех E2E-тестов

```bash
npm run e2e
```

Эта команда:

1. Запускает тестовый сервер приложения.
2. Прогоняет Cypress-тесты.
3. Прогоняет Playwright-тесты.
4. Показывает результат в терминале.

Для тестов используется порт:

```text
http://localhost:3100
```

Обычный запуск приложения использует порт:

```text
http://localhost:3000
```

Это сделано специально, чтобы тесты не конфликтовали с обычным запуском проекта.

### Что проверяют тесты

#### `library.cy.js` и `library.spec.js`

Проверяют страницу библиотеки:

- библиотека открывается;
- карточки книг отображаются;
- поле поиска видно пользователю;
- фильтр жанров доступен;
- выбор жанра показывает правильные книги;
- пункт `Все жанры` возвращает общий каталог.

#### `bookPage.cy.js` и `bookPage.spec.js`

Проверяют страницу книги:

- пользователь кликает по карточке;
- происходит переход на страницу книги;
- отображается информация о книге;
- доступна кнопка открытия читалки;
- кнопка браузера “Назад” возвращает пользователя в библиотеку.

#### `readerModal.cy.js` и `readerModal.spec.js`

Проверяют модальное окно чтения:

- читалка открывается;
- отображается первая страница;
- кнопка следующей страницы работает;
- кнопка предыдущей страницы работает;
- модальное окно закрывается.

## Частые проблемы и решения

### Проблема: `npm` не найден

Причина: Node.js не установлен или терминал был открыт до установки.

Решение:

1. Установите Node.js LTS с сайта `https://nodejs.org/`.
2. Закройте терминал.
3. Откройте терминал заново.
4. Проверьте:

```bash
node -v
npm -v
```

### Проблема: команда введена не в той папке

Если команда `npm install` пишет, что не найден `package.json`, значит терминал открыт не в корне проекта.

Решение:

1. Откройте папку `Book-ekz`.
2. Убедитесь, что там есть `package.json`.

### Проблема: порт 3000 занят

Обычно это значит, что приложение уже запущено в другом терминале.

Решение:

1. Найдите терминал, где запущен проект.
2. Нажмите `Ctrl + C`.
3. Запустите проект снова.

Для E2E-тестов используется порт `3100`, поэтому обычный запуск на `3000` и тесты меньше конфликтуют друг с другом.

### Проблема: тесты Playwright просят установить браузеры

Выполните:

```bash
npm run playwright:install
```

### Проблема: Cypress не запускается

Выполните:

```bash
npm run cypress:install
```

### Проблема: в терминале много предупреждений

Предупреждения от `Browserslist`, `webpack-dev-server` или `npm audit` не всегда означают ошибку. Если приложение открылось в браузере и сборка пишет `Compiled successfully`, проект работает.

## Как понять, что проект полностью готов к демонстрации

Перед показом дипломной работы выполните три команды:

```bash
npm install
npm run build
npm run e2e
```

Проект готов, если:

- `npm install` завершился без критической ошибки;
- `npm run build` написал `Compiled successfully`;
- `npm run e2e` показал успешное прохождение Cypress и Playwright;
- приложение открывается по адресу `http://localhost:3000/library`.

## Краткое описание для пояснительной записки

Разработанное веб-приложение ReadNext представляет собой электронную библиотеку, реализованную на React. Пользователь может просматривать каталог книг, выполнять поиск, фильтровать книги по жанрам, открывать страницу выбранной книги и читать текст во встроенном модальном окне. Приложение является одностраничным, использует маршрутизацию на стороне клиента и хранит пользовательское состояние в localStorage браузера. Для проверки пользовательских сценариев добавлены E2E-тесты на Cypress и Playwright.

## Контрольный пользовательский путь

Основной сценарий, который покрывается интерфейсом и тестами:

1. Пользователь открывает библиотеку.
2. Пользователь видит список книг.
3. Пользователь фильтрует книги по жанру.
4. Пользователь открывает книгу.
5. Пользователь переходит на страницу книги.
6. Пользователь открывает ReaderModal.
7. Пользователь листает страницы.
8. Пользователь закрывает ReaderModal.
9. Пользователь возвращается в библиотеку.
